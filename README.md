# PromptCollector

Прототип **интеллектуального испытания** на основе **соревновательной инженерии запросов** для задачи **прогнозирования временного ряда** с использованием больших языковых моделей (LLM) и автоматизированного контура оценки стратегии участника.

Проект реализует идею из статьи:
**«Модель интеллектуального испытания на основе соревновательной инженерии запросов и структурированного анализа стратегий» (А.О. Стальной)** — оценивание учитывает не только итоговый прогноз, но и качество/динамику попыток под количественной обратной связью.

## Что делает система

- **Задача**: по контексту из последних \(N\) свечей OHLCV предсказать **цены закрытия** на горизонте \(H\) шагов.
- **Интерфейс**: Telegram-бот принимает `user_prompt` (инструкцию), автоматически добавляет числовой контекст и отправляет полный запрос в LLM.
- **Оценка**: промпт тестируется на нескольких одинаковых для всех тестовых окнах \(K\) (в текущем коде \(K=3\)), после чего считаются метрики и обновляются лидерборды.
- **Семантическая валидация (опционально)**: модуль `checker.py` может отклонять нерелевантные/«несодержательные» промпты по косинусному сходству эмбеддингов.

## Метрики

В `test_test_prompt.py` для каждого окна (тестового сэмпла) считаются:

- **MAE**
- **RMSE**
- **sMAPE**
- **Direction Accuracy**

## Структура репозитория

- `main.py` — Telegram-бот: очередь запросов, тестирование промпта, сохранение результатов, лидерборды/статистика.
- `test_test_prompt.py` — класс `Tester`: сбор ответа LLM, парсинг чисел, расчёт метрик.
- `db_controller.py` — `DBController`: SQLite для свечей (`financial_data.db`), формирование тестовых сэмплов и форматирование контекста.
- `checker.py` — семантическая валидация промптов (SentenceTransformers + cosine similarity), в `main.py` сейчас отключена.
- `financial_data.db` — пример/рабочая БД со свечами (SQLite).
- `user_prompts.db` — БД результатов (SQLite), создаётся/расширяется автоматически при запуске бота.
- `amalyzer.ipynb`, `checker.ipynb`, `test_model.ipynb` — исследовательские ноутбуки (анализ, эксперименты, прототипирование).

## Быстрый старт (Windows / PowerShell)

### 1) Подготовка окружения

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

```

### 2) Конфигурация секретов

Создайте файл `private_info.json` (он **не должен** попадать в публичный репозиторий):

```json
{
  "OAUTH_TOKEN": "…",
  "FOLDER_ID": "…",
  "MODEL_URI": "…",
  "BOT_TOKEN": "…"
}
```

Использование полей зависит от выбранного бэкенда LLM.

### 3) Данные со свечами

Бот берёт данные из SQLite-файла `financial_data.db`. Он должен содержать таблицу `candles` со столбцами:
`symbol, interval, datetime, open, high, low, close, volume`.

В репозитории уже присутствует `financial_data.db` (если он актуален для ваших экспериментов — достаточно оставить как есть).

### 4) Запуск бота

```powershell
python main.py
```

После запуска:

- откройте Telegram и напишите боту команду `/start`
- отправьте свой промпт одним сообщением
- подтвердите тестирование кнопкой «✅ Да, тестировать»

## Как устроено тестирование в текущем коде

- **Инструмент/таймфрейм**: сейчас зафиксировано `BTCUSDT` и `1d` в `main.py`.
- **Горизонт**: \(H=5\) (цены закрытия следующих 5 свечей).
- **Количество окон**: `NUM_WINDOWS = 3` (берутся 3 сэмпла из `financial_data.db`).
- **Выбор окон**: псевдослучайная выборка с фиксированным seed (для воспроизводимости).

## Базы данных

### `financial_data.db`

Создаётся/поддерживается `DBController`:

- `tickers(symbol, base, quote, exchange)`
- `candles(symbol, interval, datetime, open, high, low, close, volume)`

### `user_prompts.db`

Создаётся при старте бота (`init_db()` в `main.py`):

- `prompts` — все попытки (агрегированные метрики + метрики/предсказания по окнам)
- `user_attempts` — счётчик попыток на пользователя

## Лидерборды

В боте доступна команда/кнопка «🏆 Топ промптов»:

- отдельные рейтинги по **sMAPE**, **DA**, **MAE**, **RMSE**
- **общий рейтинг** как сумма мест (dense ranking по каждой метрике в текущей логике)

## Валидация релевантности промптов

Файл `checker.py` реализует прототип семантической валидации с моделью эмбеддингов
`intfloat/multilingual-e5-base`. В `main.py` сейчас используется заглушка:

- `checked, similarity = True, 1.0`

Чтобы включить проверку, нужно вернуть вызов `Checker().check(...)` и подобрать порог под вашу предметную область/датасет.
